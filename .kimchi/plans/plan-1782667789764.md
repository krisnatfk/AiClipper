# AutoClip AI — Flow Refactor & Full Pipeline Plan

## Goal
Mengubah AutoClip AI dari flow "upload → langsung proses" menjadi flow profesional OpusClip-style: **upload/link → halaman konfigurasi → submit → loading card dengan progress asli → result clips → editor non-destructive → render ulang → delete project/clip**, dengan backend worker yang menangani AI Clipping, Don't-clip (import-only), dan re-render clip secara terpisah, tanpa watermark OpusClip.

## Constraints
- **No new dependencies** selain yang sudah ada (Next.js 14, Drizzle, Zustand, FFmpeg, faster-whisper, Gemini). Tidak boleh tambah library berat baru.
- **Branding AutoClip AI** saja — bukan OpusClip sebagai brand final.
- **Worker render lokal via FFmpeg** — output final TIDAK boleh pakai export OpusClip (spec P.20).
- **Non-destructive editor**: semua edit disimpan sebagai JSON config (clip_edits), render final hanya saat Export.
- **Tidak boleh hardcoded stuck progress** — progress harus mapping ke status asli dari worker (spec O.18).
- **Polling 10–20 detik**, bukan spam 5 detik.
- **Preserve existing functionality** — jangan hapus fitur lama sebelum flow baru jalan.
- **Direct video URL only** untuk link input; YouTube/TikTok tampilkan pesan "coming soon", jangan crash.
- Database: SQLite (local.db / Turso) dengan Drizzle ORM existing.

## Verification Strategy
End-to-end test (spec Phase 11) setelah semua chunk:

1. **Upload flow**: `npm run dev` + `npm run worker`. Upload MP4 di `/create` → redirect `/projects/:id/configure` → DB status UPLOADED, no job.
2. **Configure**: isi semua field → submit → DB status QUEUED + job PROCESS_VIDEO → redirect `/projects/:id`.
3. **Processing**: project card menunjukkan progress 15→25→40→60→70→85→100 dengan label status benar (polling 20s). Worker log via `/api/projects/:id/logs`.
4. **Direct URL**: paste direct MP4 URL → configure → start → worker download → COMPLETED.
5. **Don't clip mode**: pilih "Don't clip" → start → IMPORT_ONLY job → 1 clip full video → masuk editor.
6. **Result clips**: `/projects/:id/clips` menampilkan clips dengan score, search/filter/sort, download dari `/api/clips/:id/video`.
7. **Editor**: edit clip → Save (PATCH 200) → Export → RENDER_CLIP job → progress modal → download final.
8. **Delete clip**: ClipCard delete → modal → "delete file" → clip hilang + file output terhapus.
9. **Delete project**: dashboard card menu → delete → modal → scope "all" → project + clips + source hilang.
10. **Failed project**: stop worker, start project → status FAILED + error + Retry tampil (no stuck loading).
11. **Build check**: `npm run build` lulus tanpa type error. `npm run lint` clean.

Per-chunk verification: jalankan command relevan (curl endpoint, cek DB via `npm run db:studio`, jalankan `npm run worker:once`) setelah tiap chunk.
